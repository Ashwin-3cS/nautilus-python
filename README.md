# Nautilus Python Template

Python TEE template for AWS Nitro Enclaves on the Sui blockchain. Runs inside an enclave with Ed25519 key generation, NSM attestation, and HTTP signing — all using Python's stdlib and [pynacl](https://pynacl.readthedocs.io/).

Part of the [nautilus-ops](https://github.com/Ashwin-3cS/nautilus-ops/) ecosystem.

## How It Works

The app runs a Python HTTP server inside a Nitro Enclave. On startup, it generates an ephemeral Ed25519 keypair and exposes three endpoints:

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/health` | GET | Returns `{"status": "ok", "template": "python"}` |
| `/attestation` | GET | Returns NSM attestation document with the public key embedded |
| `/sign` | POST | Signs the request body with Ed25519, returns `{"signature": "<hex>"}` |

The signing pattern uses `blake2b256(data)` before Ed25519 signing, compatible with the on-chain `verify_signed_data<T>` function.

## Architecture

```
EC2 Instance (Nitro-enabled)
┌──────────────────────────────────────────────┐
│  socat bridge (TCP:5000 <-> VSOCK:5000)      │
│                                              │
│  ┌────────────────────────────────────────┐  │
│  │  Nitro Enclave (isolated, no network)  │  │
│  │                                        │  │
│  │  nit init (loopback, NSM, mounts)      │  │
│  │         ↓                              │  │
│  │  run.sh                                │  │
│  │    socat VSOCK:5000 → TCP:localhost     │  │
│  │    python3 app.py                      │  │
│  │      ├── /health                       │  │
│  │      ├── /attestation (NSM ioctl)      │  │
│  │      └── /sign (Ed25519)               │  │
│  └────────────────────────────────────────┘  │
└──────────────────────────────────────────────┘
```

The enclave has no network access — communication happens over VSOCK. A socat bridge on the host forwards TCP:5000 to VSOCK, and another socat inside the enclave forwards VSOCK back to the app's localhost:5000.

## Build

The enclave image is built using a multi-stage Docker build with [StageX](https://stagex.tools/) for reproducible EIF packaging:

1. **python-build** — Alpine Python 3.12 installs dependencies, collects shared libraries
2. **initramfs assembly** — Copies Python runtime, app code, socat, busybox into a minimal rootfs
3. **eif_build** — Packages kernel + initramfs into `nitro.eif` with deterministic PCR measurements

```bash
# Build locally (requires Docker)
make

# Output: out/nitro.eif, out/nitro.pcrs
```

## Deploy

```bash
# Generate CI workflow (from this directory)
nautilus init-ci

# Set GitHub secrets: TEE_EC2_HOST, TEE_EC2_USER, TEE_EC2_SSH_KEY
# Push to main → enclave deploys automatically
git push
```

## E2E Flow

Once the enclave is running:

```bash
# 1. Fetch attestation
nautilus attest --host <EC2_IP> --out pcrs.json

# 2. Update PCRs on-chain
nautilus update-pcrs --pcr-file pcrs.json

# 3. Register enclave
nautilus register-enclave --host <EC2_IP>

# 4. Verify signature
nautilus verify-signature --host <EC2_IP> --enclave-id <ENCLAVE_ID> --data "hello"
```

## NSM Implementation

The NSM (Nitro Security Module) interface communicates directly with `/dev/nsm` via ioctl, matching the kernel driver's expected layout:

- **ioctl number**: `0xC0200A00` — `_IOWR(0x0A, 0, 32)`
- **message struct**: Two `iovec` structs (pointer + uint64 length), 32 bytes total
- **request/response**: CBOR-encoded, using the `cbor2` library

Outside an enclave (`/dev/nsm` doesn't exist), a mock attestation document is returned for development.

## Requirements

- **Python 3.12** (Alpine musl-based, embedded in the EIF)
- **pynacl** >= 1.5 — Ed25519 signing
- **cbor2** >= 5.6 — CBOR encoding for NSM requests
